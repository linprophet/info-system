### 🧠 VLM / LLM Post-training

**本周一句话**: 模型发布和评估方法论齐头并进——Gemma 4 开源重磅，同时 Video-Oasis 和 DataComp-VLM 分别从评估和数据角度推动 VLM 研究的基础设施建设。

- **[Gemma 4 Technical Report](https://huggingface.co/papers/2607.02770)** — _HF Papers · score 0.95_
  Google 发布新一代开源多模态语言模型家族，涵盖 dense 和 MoE 架构（2.3B–31B），改进视觉和音频编码器，12B 模型采用 encoder-free 架构直接处理原始音频和图像 patch，集成 thinking mode，在 STEM、多模态和长上下文基准上实现显著性能跃升。
- **[Video-Oasis: Rethinking Evaluation of Video Understanding](https://huggingface.co/papers/2603.29616)** — _HF Papers · score 0.85_
  系统审计现有视频理解基准，发现 55% 的样本无需视觉输入或时间上下文即可解答，过滤后暴露了 Video-LLM 在真正视频理解上的严重不足，评估了 10+ 个模型。
- **[DataComp-VLM: Improved Open Datasets for Vision-Language Models](https://huggingface.co/papers/2606.28551)** — _HF Papers · score 0.85_
  为 VLM 训练数据策展策略提供系统基准，收集 160 个数据集（6T 多模态 token），支持 1B–8B 模型和 6.25B–200B token 预算下的策展策略测试，在 52 个下游基准上评估。
- **[Ideas Have Genomes: Benchmarking Scientific Lineage Reasoning](https://huggingface.co/papers/2607.08758)** — _HF Papers · score 0.85_
  IdeaGene-Bench 评估 AI 追踪科学思想继承、变异和重组的能力，包含 1,961 条谱系轨迹和 1,085 个 Idea Genome 对象。
- **[Unified Audio Intelligence Without Regressing on Text Intelligence](https://huggingface.co/papers/2607.05196)** — _HF Papers · score 0.85_
  NVIDIA 发布 Nemotron-Labs-Audex-30B-A3B，统一音频-文本 MoE LLM，采用单一 Transformer decoder 处理音频和文本，在 157.4B 音频 token 和 320.5B 文本 token 上训练，实现强音频-文本融合。
- **[BRAID: Bridging Interleaved Multi-Modal Reasoning as a Unified Decision Process](https://huggingface.co/papers/2607.03748)** — _HF Papers · score 0.80_
  将交错文本-图像推理建模为统一 MDP，首次让策略梯度通过完整的多模态轨迹传播，实现文本和视觉生成的联合 RL 优化。
- **[Vision as Unified Multimodal Generation](https://huggingface.co/papers/2607.06560)** — _HF Papers · score 0.75_
  SenseNova-Vision 将计算机视觉任务表达为统一多模态生成，无需任务特定架构，输出文本（符号输出）或图像（密集空间预测），构建了大规模视觉指令-响应语料库。

- **[UI-MOPD: Multi-Platform On-Policy Distillation for Continual GUI Agent Learning](https://huggingface.co/papers/2607.04425)** — _HF Papers · score 0.75_

- **[Better tools made Copilot code review worse. Here's how we actually improved it.](https://github.blog/ai-and-ml/github-copilot/better-tools-made-copilot-code-review-worse-heres-how-we-actually-improved-it/)** — _GitHub Copilot Blog · score 0.75_

### 🎯 Agent RL & Reasoning

**本周一句话**: 训练稳定性和效率成为焦点——从训练-推理不一致的根本原因分析到 on-policy 蒸馏的 token 级优化，RL 基础设施正在精细化。

- **[The Mirage of Optimizing Training Policies: Monotonic Inference Policies as the Real Objective for LLM Reinforcement Learning](https://huggingface.co/papers/2606.29526)** — _HF Papers · score 0.85_
  揭示 LLM RL 中训练-推理不一致（off-policyness）是训练不稳定的关键原因，指出现有工作目标错位，提出单调推理策略作为真正优化目标。
- **[DSpark: Confidence-Scheduled Speculative Decoding with Semi-Autoregressive Generation](https://huggingface.co/papers/2607.05147)** — _HF Papers · score 0.75_
  DeepSeek 提出半自回归投机解码框架，结合并行骨干和轻量级顺序模块，通过置信度调度和延迟记忆提高高并发服务中的吞吐量。
- **[LLM-as-a-Tutor: Policy-Aware Prompt Adaptation for Non-Verifiable RL](https://huggingface.co/papers/2607.04412)** — _HF Papers · score 0.70_
  KAIST 提出框架将 LLM 从 judge 扩展到 tutor，动态检测非挑战性 prompt 并生成更难的变体，保持判别性奖励信号。
- **[TurnOPD: Making On-Policy Distillation Turn-Aware for Efficient Long-Horizon Agent Training](https://huggingface.co/papers/2607.05804)** — _HF Papers · score 0.70_
  腾讯混元提出 turn 级预算策略，自适应截断尾部回合（弱 KL 监督）并重新平衡 token 级损失，减少长 horizon agent 训练中的资源浪费。

### 🎬 Video Generation

**本周一句话**: 实时交互式视频生成取得突破——Vidu S1 和 Wan-Streamer v0.2 分别从语音控制和分辨率提升两个方向推动实时视频生成的前沿。

- **[Vidu S1: A Real-Time Interactive Video Generation Model](https://huggingface.co/papers/2607.03118)** — _HF Papers · score 0.85_
  清华团队发布实时交互式视频生成模型，支持通过语音指令控制数字角色，实现无限长度实时视频生成（无模糊、漂移或视觉失真），在消费级 GPU 上达到 42 FPS 540p 输出。
- **[Wan-Streamer v0.2: Higher Resolution, Same Latency](https://huggingface.co/papers/2607.04443)** — _HF Papers · score 0.85_
  实时音视频交互模型升级至 640×368 分辨率，保持约 200ms 模型端信号到信号延迟，支持场景接地中景 agent 的姿势、手势和局部场景布局在实时对话中保持可读。
- **[LongE2V: Long-Horizon Event-based Video Reconstruction, Prediction, and Frame Interpolation](https://huggingface.co/papers/2607.08770)** — _HF Papers · score 0.65_
  利用预训练视频扩散先验处理事件流视频重建、预测和帧插值，引入自回归展开和自适应上下文切换缓解长序列时间漂移。

### 🌍 World Models

**本周一句话**: 世界模型从单玩家扩展到多玩家交互，同时向机器人策略评估和数字遥操作两个应用方向深化。

- **[GigaWorld-1: A Roadmap to Build World Models for Robot Policy Evaluation](https://huggingface.co/papers/2607.02642)** — _HF Papers · score 0.85_
  系统研究世界模型作为机器人策略评估替代方案，构建 WMBench 基准（真实遥操作数据 + 匹配策略 rollout），分析模型家族、动作编码和 rollout 时长对评估可靠性的影响。
- **[AlayaWorld: Long-Horizon and Playable Video World Generation](https://huggingface.co/papers/2607.06291)** — _HF Papers · score 0.80_
  全栈开源可玩视频世界模型，自回归合成未来观察，支持用户交互，在游戏录制和真实世界视频上训练，捕捉多样视觉外观和物理动力学。
- **[RynnWorld-4D: 4D Embodied World Models for Robotic Manipulation](https://huggingface.co/papers/2607.06559)** — _HF Papers · score 0.75_
  提出 RGB-DF（RGB + 深度 + 光流）作为物理接地表示，联合生成未来 RGB 帧、深度图和光流，缩小世界预测与策略学习之间的差距。
- **[RynnWorld-Teleop: An Action-Conditioned World Model for Digital Teleoperation](https://huggingface.co/papers/2607.06558)** — _HF Papers · score 0.75_
  用生成世界模型替代真实机器人，操作员手部姿态流驱动模型合成高保真第一人称视频，实现与物理硬件解耦的数据收集。
- **[Multiplayer Interactive World Models with Representation Autoencoders](https://huggingface.co/papers/2607.05352)** — _HF Papers · score 0.75_
  首个多玩家世界模型，在 Rocket League 游戏中基于多个 agent 的动作流条件生成，5B 参数潜在扩散模型在单张 B200 GPU 上以 20 FPS 实时生成四玩家对局。

### 🎨 Image Generation

**本周一句话**: 3D 场景生成和量化方法取得进展——PixWorld 统一了 3D 重建与生成，OrbitQuant 实现了数据无关的 DiT 量化。

- **[PixWorld: Unifying 3D Scene Generation and Reconstruction in Pixel Space](https://huggingface.co/papers/2607.05373)** — _HF Papers · score 0.70_
  在像素空间扩散范式下统一 3D 场景重建和生成，直接在渲染图像上监督扩散，避免潜在编码的信息损失和预训练 VAE 依赖。
- **[OrbitQuant: Data-Agnostic Quantization for Image and Video Diffusion Transformers](https://huggingface.co/papers/2607.02461)** — _HF Papers · score 0.70_
  提出数据无关的权重量化方法，在归一化旋转基中量化，单个 Lloyd-Max 码本适用于所有时间步、prompt 和层，无需重新拟合校准数据。
- **[Canvas360: Enhancing In-context Panoramic Generation via Geometric-aware Pretraining](https://huggingface.co/papers/2607.08765)** — _HF Papers · score 0.70_
  两阶段全景生成框架，包含 1M 高质量配对全景样本数据集，通过并行深度生成、速度循环填充和相似性损失正则化学习几何感知表示。
